## UC-UI-01：通过 CLI 执行备份

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-UI-01 |
| **用例名称** | 通过 CLI 执行备份 |
| **参与者** | 用户 |
| **前置条件** | backer-cli 已编译；源目录与目标仓库路径可用 |
| **后置条件** | 执行底层备份流程（UC-TOP-01/02），输出备份结果到 stdout |
| **基本事件流** | ① 用户在终端输入 `backer backup <source> <dest> [OPTIONS]` → ② CLI11 解析参数 → ③ 构建 BackupOptions → ④ 调用 `handleBackup()` → ⑤ 输出结果 |
| **备选事件流** | 2a. 参数缺失/格式错误 → CLI11 报错并打印用法；4a. --password 未提供 → 进入交互式密码提示 |
| **包含/扩展** | 包含 UC-TOP-01（目录镜像备份）或 UC-TOP-02（打包归档备份）；扩展 UC-UI-09（设置筛选条件，通过 --include-*/--exclude-* 参数） |
| **优先级** | 高 |

## UC-UI-05：通过 GUI 执行备份

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-UI-05 |
| **用例名称** | 通过 GUI 执行备份 |
| **参与者** | 用户 |
| **前置条件** | backer-gui 已启动并显示主窗口 |
| **后置条件** | 备份完成，日志输出到文本框 |
| **基本事件流** | ① 用户切换到「备份」标签页 → ② 选择源/目标目录 → ③ 可选配置打包/压缩/加密/筛选 → ④ 点击「开始备份」→ ⑤ BackupWorker 在后台线程执行 → ⑥ 进度条实时更新 → ⑦ 完成后显示结果 |
| **备选事件流** | 5a. 用户点击「取消」→ cancelled_ 标志置位 → 当前操作中止；3a. 勾选筛选→弹出 FilterDialog 编辑条件 |
| **包含/扩展** | 包含 UC-TOP-01/02；扩展 UC-UI-09（筛选）、UC-UI-10（取消）、UC-UI-11（恢复默认） |
| **优先级** | 高 |

## UC-UI-10：取消正在进行的操作

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-UI-10 |
| **用例名称** | 取消正在进行的操作 |
| **参与者** | 用户 |
| **前置条件** | BackupWorker 正在运行备份或还原 |
