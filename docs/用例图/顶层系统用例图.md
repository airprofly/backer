# 顶层系统用例描述

## UC-TOP-01：执行目录镜像备份

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-TOP-01 |
| **用例名称** | 执行目录镜像备份 |
| **参与者** | 用户 |
| **前置条件** | 源目录存在；目标仓库目录存在或可创建 |
| **后置条件** | 目标仓库目录下生成 `{源目录名}_{YYYYMMDD_HHMMSS}` 快照子目录，内含源目录完整副本 |
| **基本事件流** | ① 用户输入 `backup <source> <dest>` 或在 GUI 备份页选择源/目标目录 → ② 系统验证源路径有效性 → ③ 系统递归遍历源目录树 → ④ 系统在目标仓库中生成带时间戳的快照目录 → ⑤ 系统逐个复制文件目录（含元数据、特殊文件） → ⑥ 系统输出备份统计（文件数/大小/耗时） |
| **备选事件流** | 2a. 源路径不存在 → 报错退出；3a. 遍历遇权限不足 → 跳过并记录警告；5a. 单个文件写入失败 → 跳过并继续；6a. 磁盘满 → 报错退出 |
| **包含/扩展** | 无 |
| **优先级** | 高 |

## UC-TOP-02：执行打包归档备份

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-TOP-02 |
| **用例名称** | 执行打包归档备份 |
| **参与者** | 用户 |
| **前置条件** | 源目录存在；目标仓库目录存在或可创建 |
| **后置条件** | 目标仓库下生成 `{源目录名}_{YYYYMMDD_HHMMSS}.tar\|zip` 归档文件 |
| **基本事件流** | ① 用户输入 `backup <source> <dest> --pack <tar\|zip>` 或在 GUI 勾选打包 → ② 系统验证源目录 → ③ 系统遍历源目录树并应用筛选 → ④ 系统将条目打包为 Tar/Zip 归档 → ⑤ 系统写入目标归档文件 → ⑥ 输出统计 |
| **备选事件流** | 同 UC-TOP-01；4a. 打包过程遇不支持的条目类型 → 跳过并警告；5a. 归档写入失败 → 清除部分产物 |
| **包含/扩展** | 无 |
| **优先级** | 高 |

## UC-TOP-03：执行还原操作

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-TOP-03 |
| **用例名称** | 执行还原操作 |
| **参与者** | 用户 |
| **前置条件** | 备份源（快照目录或归档文件）存在；还原目标目录存在或可创建 |
| **后置条件** | 还原目标下生成 `{备份源名}_R{YYYYMMDD_HHMMSS}` 目录，含恢复后的文件 |
| **基本事件流** | ① 用户输入 `restore <source> <dest>` 或在 GUI 还原页操作 → ② 系统验证备份源 → ③ 目录模式：遍历备份源目录树 → ④ 逐个还原文件/目录/特殊文件 → ⑤ 恢复元数据（权限/时间戳/属主） → ⑥ 输出统计 |
| **备选事件流** | 2a. 备份源不存在 → 报错；3a. 归档模式 → 先解压/解密再解包；4a. 类型不匹配 → 跳过并警告；5a. 恢复属主因非 root 失败 → 跳过属主恢复其余继续 |
| **包含/扩展** | 无 |
| **优先级** | 高 |

## UC-TOP-04：管理定时备份任务

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-TOP-04 |
| **用例名称** | 管理定时备份任务 |
| **参与者** | 用户 |
| **前置条件** | 系统已安装并可执行 |
| **后置条件** | 定时任务的配置被添加/删除/启用/禁用，已持久化到 `~/.config/backer/schedule.json` |
| **基本事件流** | ① 用户输入 `schedule add/list/remove/enable/disable` 或在 GUI 定时页操作 → ② 系统解析命令参数 → ③ 系统操作 ScheduleStore JSON 文件 → ④ 输出操作结果 |
| **备选事件流** | 1a. 参数不完整 → 打印用法；3a. JSON 文件损坏 → 报错退出；3b. 任务 ID 不存在 → 报错 |
| **包含/扩展** | 无 |
| **优先级**  | 高 |

## UC-TOP-05：启动守护进程定时执行备份

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-TOP-05 |
| **用例名称** | 启动守护进程定时执行备份 |
| **参与者** | 用户 |
| **前置条件** | 至少已通过 UC-TOP-04 添加了一个定时任务 |
| **后置条件** | 守护进程持续运行，到期自动执行备份并做快照淘汰 |
| **基本事件流** | ① 用户输入 `daemon` → ② 系统加载 schedule.json 中的任务列表 → ③ 计算每个任务的下次触发时间 → ④ 阻塞等待最早到期的任务 → ⑤ 任务到期 → 执行完整备份流程 → ⑥ 执行快照淘汰（如有配置） → ⑦ 重新计算下次触发时间 → 循环 ④—⑦ |
| **备选事件流** | 2a. 无已配置任务 → 报错退出；5a. 备份执行失败 → 记录日志继续等待下次触发；6a. 淘汰遇文件删除失败 → 跳过并警告；7a. Ctrl+C 收到 SIGINT → 安全退出 |
| **包含/扩展** | 包含 UC-TOP-01（执行目录镜像备份）；包含 UC-TOP-03（执行还原操作，当 retore 被调度时）；扩展 UC-SC-09/10（可选快照淘汰） |
| **优先级** | 中 |
