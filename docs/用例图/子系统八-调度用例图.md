# 子系统八：调度用例描述

参与者：用户（CRUD 操作者）、系统时钟（定时信号源）、守护进程（后台循环）

---

## UC-SC-01：添加定时备份任务

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-SC-01 |
| **用例名称** | 添加定时备份任务 |
| **参与者** | 用户 |
| **前置条件** | 无 |
| **后置条件** | 新任务被追加到 schedule.json 并持久化 |
| **基本事件流** | ① CLI: `schedule add <name> <cron> <src> <dest> [OPTIONS]` 或 GUI 定时页「添加」→ ② 解析 cron 表达式、源/目标路径、压缩/加密/淘汰等可选参数 → ③ 生成 job_id（基于 name） → ④ `ScheduleStore::load()` 加载已有任务 → ⑤ 追加新任务 → ⑥ `ScheduleStore::save()` 持久化 |
| **备选事件流** | 4a. JSON 文件损坏 → 返回错误；5a. ID 冲突 → 自动追加数字后缀去重 |
| **包含/扩展** | 包含 UC-SC-11（持久化保存） |
| **优先级** | 高 |

## UC-SC-05：守护进程加载任务配置

| 字段 | 内容 |
|------|------|

| **用例名称** | 守护进程加载任务配置 |
| **参与者** | 守护进程 |

## UC-SC-07：阻塞等待到期触发

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-SC-07 |
| **用例名称** | 阻塞等待到期触发 |
| **参与者** | 系统时钟 |
| **前置条件** | 调度器已初始化且 `scheduler.run()` 被调用 |
| **后置条件** | 线程阻塞在 `cv_.wait_until(nextFire)` 直到到期或 stop 被调用 |
| **基本事件流** | ① `run()` 循环 → ② 加锁 → ③ `earliestNext()` 找最早→ ④ `cv_.wait_until(lock, nextFire)` → ⑤ 被唤醒 → ⑥ `fireDueJobs(now)` → ⑦ 重新计算 → ⑧ 回到 ② |
| **备选事件流** | 3a. 无已启用任务 → wait 到 max()；5a. stop 请求 → 退出循环；5b. spurious wakeup → 重新检查 |
| **包含/扩展** | 包含 UC-SC-06（计算时间）、UC-SC-08（执行到期任务） |
| **优先级** | 高 |

## UC-SC-08：执行到期备份任务

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-SC-08 |
| **用例名称** | 执行到期备份任务 |
| **参与者** | 调度器（executor 回调） |
| **前置条件** | 当前时间 >= 某任务的 nextFireTime |
| **后置条件** | 备份已执行；快照目录已创建；旧快照已被淘汰（如有配置） |
| **基本事件流** | ① `fireDueJobs(now)` → ② 遍历找到 ≤ now 的任务 → ③ `makeSnapshotPath(job.destination)` 生成带时间戳快照目录 → ④ `executor_(job)` 回调 → ⑤ 回调内部调用 `handleBackup(job.source, snapshotDir, opts)` → ⑥ 执行 RetentionPolicy → ⑦ 返回 true |
| **备选事件流** | 4a. 备份失败 → 返回 false → 日志记录 |
| **包含/扩展** | 包含 UC-TOP-01（执行备份）；扩展 UC-SC-09/10（可选淘汰） |
| **优先级** | 高 |

| **用例名称** | 按数量淘汰快照 |
| **参与者** | 调度器（executor 回调） |

| **包含/扩展** | 扩展 UC-SC-08 |
| **优先级** | 中 |
