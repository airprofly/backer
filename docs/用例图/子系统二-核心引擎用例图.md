# 子系统二：核心引擎用例描述

参与者：CLI/GUI（表现层调用者）

## UC-CE-01：验证源路径有效性

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CE-01 |
| **用例名称** | 验证源路径有效性 |
| **参与者** | BackupEngine |
| **前置条件** | 用户已提供 source 路径 |
| **后置条件** | 确认路径存在且为目录，否则返回 ErrorCode::kPathNotFound |
| **基本事件流** | ① 检查 `std::filesystem::exists(source)` → ② 检查 `is_directory(source)` |
| **备选事件流** | 1a. 路径不存在 → 返回错误；2a. 路径不是目录 → 返回错误 |
| **包含/扩展** | 被 UC-CE-04/05 包含 |
| **优先级** | 高 |

## UC-CE-02：遍历源目录树收集文件条目

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CE-02 |
| **用例名称** | 遍历源目录树收集文件条目 |
| **参与者** | BackupEngine/RestoreEngine |
| **前置条件** | 源路径已通过有效性验证 |
| **后置条件** | 返回 `vector<FileEntry>` 包含每个文件/目录的相对路径、类型、大小、元数据 |
| **基本事件流** | ① 调用 `FSAbstraction::walk(root)` → ② 递归遍历所有子目录 → ③ 通过 lstat 检测每个条目的 FileType → ④ 收集 Metadata（uid/gid/mode/atim/mtim） → ⑤ 对符号链接记录 target → ⑥ 对设备文件记录 deviceMajor/Minor |
| **备选事件流** | 3a. 遇符号链接环 → 返回 kSymlinkLoop；3b. 遇权限不足的目录 → 跳过并记录警告 |
| **包含/扩展** | 被 UC-CE-04/05 包含；使用 UC-FS-01、UC-FS-08、UC-FS-09 |
| **优先级** | 高 |

## UC-CE-03：筛选文件条目

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CE-03 |
| **用例名称** | 筛选文件条目 |
| **参与者** | BackupEngine |
| **前置条件** | 已完成目录遍历得到文件列表 |
| **后置条件** | 返回筛选后的文件子集 |

| **包含/扩展** | 被 UC-CE-04/05 包含；为 UC-FL-07 的调用者 |
| **优先级** | 高 |

## UC-CE-04：执行目录镜像复制

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CE-04 |
| **用例名称** | 执行目录镜像复制 |
| **参与者** | CLI/GUI |
| **前置条件** | 源/目标有效；packer 为 nullptr |
| **后置条件** | 目标目录下逐个复制文件/目录/特殊文件，含完整元数据 |
| **基本事件流** | ① 验证源路径 → ② walk 遍历 → ③ 可选 filter 筛选 → ④ 创建目标根目录 → ⑤ 遍历每个 FileEntry：按类型 mkdir/write/symlink/mkfifo/mknod → ⑥ 恢复非目录条目的元数据 → ⑦ 目录元数据延后恢复（深→浅排序） → ⑧ 恢复根目录元数据 → ⑨ 输出统计 |
| **备选事件流** | 5a. 单个文件写入失败 → skip++ 继续；6a. 元数据恢复失败（非 root 无 CAP_CHOWN）→ 跳过属主其他继续 |
| **包含/扩展** | 包含 UC-CE-01/02/03/06/07/08；扩展 UC-CE-09（跳过失败） |
| **优先级** | 高 |

## UC-CE-05：执行打包归档写入

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CE-05 |
| **用例名称** | 执行打包归档写入 |
| **参与者** | CLI/GUI |
| **前置条件** | packer 不为 nullptr |
| **后置条件** | 目标路径下写入一个完整的归档文件 |
| **基本事件流** | ① 验证源路径 → ② walk 遍历 → ③ 可选 filter → ④ packer->pack(entries, fs, source, archive) → ⑤ fs->write(dest, archiveData) |
| **备选事件流** | 4a. pack 失败（如 Zip 格式不支持的特殊文件）→ 返回错误 |
| **包含/扩展** | 包含 UC-CE-02/03/06/08；扩展 UC-CE-09 |
| **优先级** | 高 |

## UC-CE-06：按类型创建文件/目录/特殊文件

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CE-06 |
| **用例名称** | 按类型创建文件/目录/特殊文件 |
| **参与者** | BackupEngine/RestoreEngine |
| **前置条件** | 已有 FileEntry 条目和映射后的目标路径 |
| **后置条件** | 根据 FileType 在目标路径创建对应类型的文件系统对象 |
| **基本事件流** | switch(entry.type): kRegular→fs->write 内容；kDirectory→fs->mkdir；kSymlink→createSymlink；kFifo→createFifo；kBlockDevice/kCharDevice→createDevice；kSocket→跳过 |
| **包含/扩展** | 被 UC-CE-04/05 包含 |
| **优先级** | 高 |

## UC-CE-07：恢复文件元数据

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CE-07 |
| **用例名称** | 恢复文件元数据 |
| **参与者** | BackupEngine/RestoreEngine |
| **前置条件** | 文件/目录已在目标路径创建 |
| **后置条件** | 文件的属主（uid/gid）、权限（mode）、时间戳（atime/mtime）已被恢复 |
| **基本事件流** | ① lchown() 恢复属主 → ② fchmodat() 恢复权限 → ③ utimensat() 恢复 atime/mtime 纳秒精度 |
| **备选事件流** | 1a. 非 root 且 restoreOwnership=true → 跳过属主改用记录值；2a. Linux 对符号链接跳过 chmod |
| **包含/扩展** | 被 UC-CE-04 包含；使用 UC-FS-10 |
| **优先级** | 高 |

## UC-CE-09：跳过失败文件继续执行

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CE-09 |
| **用例名称** | 跳过失败文件继续执行 |
| **参与者** | BackupEngine/RestoreEngine |
| **前置条件** | 处理单个文件/目录时发生错误 |
| **后置条件** | 当前文件被跳过，skipped 计数 +1，引擎继续处理下一个条目 |
| **基本事件流** | ① 读取/写入失败 → ② spdlog::warn 记录错误信息 → ③ continue 到下一个条目 → ④ 最终输出报告中包含 skipped 计数 |
| **包含/扩展** | 扩展 UC-CE-04/05（异常路径） |
| **优先级** | 高 |

## UC-CE-10：自动命名快照/还原目录

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CE-10 |
| **用例名称** | 自动命名快照/还原目录 |
| **参与者** | CLI/GUI |
| **前置条件** | 已确定源路径和目标仓库路径 |

| **优先级** | 中 |
