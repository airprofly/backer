# 子系统五：打包用例描述

参与者：BackupEngine（打包）、RestoreEngine（解包）

---

## UC-PK-01：打包到 Tar ustar 格式

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-PK-01 |
| **用例名称** | 打包到 Tar ustar 格式 |
| **参与者** | BackupEngine |
| **前置条件** | 文件条目列表已完成；源路径已知 |
| **后置条件** | 输出流中包含完整 POSIX ustar 格式归档，以两个全零 512B 块结尾 |
| **基本事件流** | ① `TarPacker::pack(files, fs, sourceRoot, output)` → ② 每个 FileEntry 逐条编码 → ③ `encodeHeader()` 将路径/类型/权限/设备号等写入 512B ustar 头部 → ④ 计算并写入头部校验和 → ⑤ 写入文件数据（512B 对齐） → ⑥ 写入结束标记 → ⑦ `flush()` |
| **备选事件流** | 2a. 路径超过 255 字符 → splitPath 使用 prefix 字段拆分；2b. 文件读取失败 → 返回 kReadFailed |
| **包含/扩展** | 包含 UC-PK-05（头部编码元数据）、UC-PK-07（校验和）、UC-PK-08（工厂） |
| **优先级** | 高 |

## UC-PK-02：从 Tar ustar 解包

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-PK-02 |
| **用例名称** | 从 Tar ustar 解包 |
| **参与者** | RestoreEngine |
| **前置条件** | .tar 文件存在且可读 |
| **后置条件** | 解包出的文件和目录写入目标路径（含元数据） |
| **基本事件流** | ① `TarPacker::unpack(input, dest, fs)` → ② 重复读 512B 头部 → ③ `verifyChecksum()` 验证 → ④ `decodeHeader()` 解析 → ⑤ 根据 typeflag 创建对应文件类型 → ⑥ 读取数据块 → ⑦ 全零块结束 |
| **备选事件流** | 3a. 校验和无效 → 返回 kInvalidArchive；6a. 数据读取不完整 → 报错 |
| **包含/扩展** | 包含 UC-PK-07（校验和验证）、UC-PK-08（工厂） |
| **优先级** | 高 |

| **优先级** | 高 |

## UC-PK-05：在 Tar 头部编码元数据

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-PK-05 |
| **用例名称** | 在 Tar 头部编码元数据 |
| **参与者** | TarPacker |
| **前置条件** | 已有 FileEntry 条目 |
| **后置条件** | TarHeader 的 mode/uid/gid/mtime/uname/gname/devmajor/devminor 字段已填充 |
| **基本事件流** | ① `encodeHeader(header, entry)` → ② formatOctal 填充各字段 → ③ typeflag 映射（'0'=文件, '5'=目录, '1'=硬链, '2'=符号链, '3'=字符, '4'=块, '6'=FIFO） → ④ setChecksum 计算校验和 |
| **包含/扩展** | 被 UC-PK-01 包含 |
| **优先级** | 高 |

## UC-PK-06：在 Zip 中嵌入 `.backer_zip_meta`

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-PK-06 |
| **用例名称** | 在 Zip 中嵌入 `.backer_zip_meta` |
| **参与者** | ZipPacker |
| **前置条件** | 打包/解包操作 |
| **后置条件** | 归档根目录下存在 `.backer_zip_meta` 条目（打包时）；或已从中读取元数据（解包时） |
| **基本事件流** | 打包：`buildMetaJson(files)` 生成 JSON（含每个条目的 type/size/metadata/symlinkTarget/devMajor/Minor）→ 写入 Zip；解包：`parseMetaEntries(json)` 解析 → `parseEntryMeta()` 恢复每个条目的 FullEntryMeta |
| **包含/扩展** | 被 UC-PK-03/04 包含 |
| **优先级** | 高 |

## UC-PK-07：验证 Tar 头部校验和

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-PK-07 |
| **用例名称** | 验证 Tar 头部校验和 |
| **参与者** | TarPacker |
| **前置条件** | TarHeader 已填充 |
| **后置条件** | 校验和字段被设置为正确值（打包时），或已验证通过（解包时） |
| **基本事件流** | ① `setChecksum(header)` → ② 将 chksum 字段写为空格 → ③ 累加 512B → ④ formatOctal 写入计算结果；`verifyChecksum(header)` → ⑤ 同样方法重新计算 → ⑥ 比较 |
| **包含/扩展** | 被 UC-PK-01/02 包含 |
| **优先级** | 高 |
