## UC-FL-01：按路径 GLOB 包含/排除

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-FL-01 |
| **用例名称** | 按路径 GLOB 包含/排除 |
| **参与者** | BackupEngine（通过 CriteriaFilter） |
| **前置条件** | `--include-path` 或 `--exclude-path` 已提供 |
| **后置条件** | 路径字段添加到 FilterCriteria：`c.pathGlob = pattern`；exclude 标志设置 |
| **基本事件流** | ① CLI 解析多个 --include-path GLOB → ② 每个 pattern 创建一个 FilterCriteria(pathGlob=pattern) → ③ apply 时用 fnmatch + FNM_PATHNAME 匹配相对路径 |
| **包含/扩展** | 被 UC-FL-07 包含 |
| **优先级** | 中 |

## UC-FL-07：多维度组合筛选

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-FL-07 |
| **用例名称** | 多维度组合筛选 |
| **参与者** | BackupEngine |
| **前置条件** | `buildFilter(options)` 构造了 CriteriaFilter（至少一个 criteria） |
| **后置条件** | 返回的 subset 满足全部 include 条件且不匹配任何 exclude |
| **基本事件流** | ① `CriteriaFilter::apply(files)` → ② **Include 阶段**：同维度 OR（任一通过即维度通过），异维度 AND（所有维度必须通过） → ③ **Exclude 阶段**：匹配任一 exclude 则移除 → ④ 返回剩余条目 |
| **备选事件流** | 2a. 无 include criteria → 全量进入 exclude 阶段；2b. 全量和 exclude 都无 → 返回全量（等效 NoopFilter） |
| **包含/扩展** | 包含 UC-FL-01/02/03；扩展 UC-FL-04/05/06 |
| **优先级** | 高 |

## UC-FL-08：无筛选全量备份

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-FL-08 |
| **用例名称** | 无筛选全量备份 |
| **参与者** | BackupEngine |
| **前置条件** | 用户未提供任何 --include-*/--exclude-* 选项 |
| **后置条件** | 所有遍历到的文件条目均通过 |
| **基本事件流** | ① `buildFilter()` 检测 criteria 为空 → ② 返回 `make_unique<NoopFilter>()` → ③ `NoopFilter::apply(files)` → 直接返回入参 |
| **包含/扩展** | 无 |
| **优先级** | 高 |
