# 子系统三：压缩用例描述

参与者：handleBackup/handleRestore（CLI 命令处理器）

---

## UC-CP-01：选择压缩算法

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CP-01 |
| **用例名称** | 选择压缩算法 |
| **参与者** | handleBackup/handleRestore |
| **前置条件** | 用户已通过 --compress/--decompress 参数指定算法 |
| **后置条件** | 由 UC-CP-09 工厂创建对应的 Compressor 实例 |
| **基本事件流** | ① CLI 解析 --compress gzip|zstd|lzma → ② 校验算法名称有效性 → ③ `buildCompressor(name, level)` 创建实例 |
| **备选事件流** | 1a. 算法名无效（如 --compress bzip2）→ CLI 校验函数返回错误信息 |
| **包含/扩展** | 包含 UC-CP-09（工厂创建） |
| **优先级** | 高 |

## UC-CP-02：设置压缩级别

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CP-02 |
| **用例名称** | 设置压缩级别 |
| **参与者** | handleBackup |
| **前置条件** | --compress 已指定，--compress-level 可选 |
| **后置条件** | 级别参数传入 `buildCompressor(name, level)`，0 表示使用算法默认值 |
| **基本事件流** | ① 解析 --compress-level N → ② 检 N 在算法支持的范围内 → ③ 传递给 Compressor 构造 |
| **备选事件流** | 1a. 未指定 → level=0 使用算法默认（gzip 6/zstd 3/lzma 6）；1b. 超出范围 → `isValidLevel()` 返回 false |
| **包含/扩展** | 扩展 UC-CP-03/04/05；包含 UC-CP-09 |
| **优先级** | 中 |

## UC-CP-03：gzip 压缩

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-CP-03 |
| **用例名称** | gzip 压缩 |
| **参与者** | Compressor |
| **前置条件** | 已通过 buildCompressor("gzip", level) 创建 GzipCompressor |
| **后置条件** | 压缩后的数据附加到 output vector，文件后缀 `.gz` |
| **基本事件流** | ① `GzipCompressor::compress(input, output)` → ② 内部调用 zlib deflateInit2 → ③ 边压缩边写入 output → ④ deflateEnd 清理 |
| **包含/扩展** | 包含 UC-CP-01；扩展 UC-CP-02 |
| **优先级** | 高 |

## UC-CP-09：工厂创建压缩器

| 字段 | 内容 |
|------|------|
