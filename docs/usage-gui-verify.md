# Backer GUI 功能验证手册

## 前提：启动 GUI

```bash
# 回到项目根目录
cd ~/projects/backer

# 编译（若之前编译过会增量编译，很快）
cmake --build build -j$(nproc)

# 启动 GUI（脚本会自动设置 LD_LIBRARY_PATH）
bash scripts/run-gui.sh
```

或者手动跑：
```bash
export LD_LIBRARY_PATH="$HOME/.local/backer-deps/usr/lib/x86_64-linux-gnu:$LD_LIBRARY_PATH"
./build/backer-gui
```

启动后你会看到 4 个标签页：**备份** | **还原** | **定时** | **设置**

---

## 验证项 ①：基本备份功能

**目标**：确认最简备份流程能跑通

1. 在「源目录」点「浏览」→ 选 `~/projects/backer/scripts/` 作为源
2. 在「目标目录」点「浏览」→ 选 `~/Desktop/backup-test/`（或其他空目录）作为输出
3. 不勾选 打包/压缩/加密/筛选 → 点「**开始备份**」
4. 观察进度条和日志输出 → 等待完成
5. ✅ 预期：日志显示「备份完成」，进度 100%

---

## 验证项 ②：Tar 打包备份

**目标**：确认 Tar/Zip 打包格式生效

1. 源目录选 `~/projects/backer/scripts/`
2. 目标目录选 `~/Desktop/backup-test2/`
3. **勾选「打包」** → 下拉选 **Tar**
4. 点「开始备份」
5. ✅ 预期：目标目录下生成 `.tar` 文件（而非散文件）
6. 再试一次选 **Zip** → ✅ 预期：生成 `.zip` 文件

---

## 验证项 ③：压缩功能（gzip / zstd / lzma）

**目标**：确认三种压缩算法正常工作

1. 源目录选 `~/projects/backer/`
2. 目标目录选 `~/Desktop/backup-test3/`
3. 勾选「打包」→ Tar
4. **勾选「压缩」** → 下拉选 **gzip** → 级别调为 **6**
5. 点「开始备份」
6. ✅ 预期：最终产物是 `.tar.gz` 文件
7. 分别再试 **zstd**（产物 `.tar.zst`）和 **lzma**（产物 `.tar.xz`）

---

## 验证项 ④：加密功能（AES-256 / SM4）

**目标**：确认加密 + 解密还原完整链路跑通

### 加密备份

1. 源目录选 `~/projects/backer/scripts/`
2. 目标目录选 `~/Desktop/backup-enc-test/`
3. 勾选「打包」→ Tar
4. **勾选「加密」** → 选 **AES-256**
5. 输入密码：`test123` → 确认密码：`test123`
6. 点「开始备份」
7. ✅ 预期：生成 `.tar.enc` 文件

### 还原加密备份

1. 切换到「**还原**」标签页
2. 「备份源」→ 浏览选刚才的 `backup-enc-test/` 目录（或里面的 `.tar.enc` 文件）
3. 「还原到」→ 选 `~/Desktop/restore-enc-test/`
4. **勾选「解密」** → 选 **AES-256** → 输密码 `test123` → 确认密码 `test123`
5. 点「开始还原」
6. ✅ 预期：还原成功，`restore-enc-test/` 下有原文件

### 还原解密密码验证

1. 勾选「解密」→ 密码输 `123`，确认密码输 `456`
2. 点「开始还原」
3. ✅ 预期：弹窗提示「两次输入的密码不一致」，不执行还原

### 再测试 SM4

1. 同上面的步骤，加密选 **SM4** → 还原选 **SM4**
2. ✅ 预期：SM4 加密还原同样成功

## 验证项 ⑤：筛选器（6 维度）

**目标**：确认路径/类型/时间/大小/所有者筛选全部生效

1. 源目录选 `~/projects/backer/src/`（这个目录文件种类丰富）
2. 目标目录选 `~/Desktop/backup-filter-test/`
3. **勾选「使用筛选」** → 点「**编辑筛选条件**」

在弹出的对话框中：

### 5a. 路径筛选
- 点「+」添加包含路径：`*cli*` → ✅ 预期：只备份 cli 目录下的文件

### 5b. 类型筛选
- 取消「文件」和「目录」之外的所有勾 → ✅ 预期：只备份普通文件和目录，跳过特殊文件

### 5c. 时间筛选
- 勾选「启用时间筛选」→ 设置"从"为 7 天前 → ✅ 预期：只备份 7 天内有修改的文件

### 5d. 大小筛选
- 勾选「启用大小筛选」→ 最小 1024 B → ✅ 预期：只备份大于 1KB 的文件

### 5e. 所有者筛选
- 在「所有者」输入当前用户名（`whoami`）→ ✅ 预期：只备份该用户的文件

4. 点「确定」关闭对话框
5. ✅ 页面上的「包含路径」等文本框中会显示你刚设置的值
6. 点「开始备份」→ ✅ 预期：按筛选条件备份

---

## 验证项 ⑥：还原功能（非加密）

**目标**：确认普通还原流程

1. 先用 CLI 打一个普通 tar 包：
   ```bash
   ./build/backer-cli backup ~/projects/backer/scripts/ /tmp/verify-source.tar --pack tar
   ```
2. 切到 GUI「还原」标签页
3. 「备份源」→ 选生成的 `/tmp/verify-source.tar`
4. 「还原到」→ 选 `~/Desktop/verify-restore/`
5. 勾选「打包格式」→ Tar
6. ✅ 「保留元数据」和「处理特殊文件」默认勾选
7. 点「开始还原」
8. ✅ 预期：还原成功

---

## 验证项 ⑧：设置页面

**目标**：确认设置可修改

1. 切换到「**设置**」标签页
2. ✅ 预期：看到默认压缩算法、压缩级别、日志级别等选项

## 验证项 ⑨：恢复默认按钮

**目标**：确认备份页和还原页的「恢复默认」按钮正常

1. 在「**备份**」标签页中填一些内容（源目录、勾选加密、输密码等）
2. 点左下角「**恢复默认**」
3. ✅ 预期：源目录、目标目录清空，打包/压缩/加密/筛选的勾选取消，密码框清空
4. 切换到「**还原**」标签页，同样填写一些内容后点「恢复默认」
5. ✅ 预期：所有字段恢复初始状态

---

## 验证项 ⑩：取消操作

**目标**：确认可中断正在进行的任务

1. 选一个大目录作为源（如 `~/projects/backer/`）
2. 点「开始备份」
3. 立刻点「**取消**」
4. ✅ 预期：任务中断，日志显示「正在取消...」
