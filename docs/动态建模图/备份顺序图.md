# 图名

系统场景顺序图——备份流程

## 概述

本顺序图描述了 Backer 备份还原工具的核心业务场景——用户通过 CLI 命令行执行一次完整目录备份的全过程。覆盖从用户输入命令开始，经 CLI11 参数解析、handleBackup 预处理、BackupEngine 引擎调度、文件系统遍历与筛选过滤，到目录镜像或归档打包两条主要执行路径，以及压缩与加密两组可选后处理阶段，最终输出备份统计信息的完整交互序列。涉及 CLI 命令层、引擎调度层、文件系统抽象层、筛选器、打包器、压缩器、加密器和存储层共 8 个主要模块。

## 参与者列表

| 参与者 | 类型 | 职责 | 对应源码 |
|--------|------|------|----------|
| 用户 | actor | CLI 操作者，输入备份命令并接收结果 | — |
| CLI | participant | 命令行参数解析与子命令分发 | main.cpp → CLI11 解析 |
| handleBackup | participant | 备份命令总协调器，编排预处理和后处理 | commands.cpp → handleBackup() |
| BackupEngine | participant | 核心备份引擎，调度 walk→filter→pack/mirror 流水线 | core/backup_engine.h → engine.backup() |
| FSAbstraction | participant | 文件系统抽象层，提供遍历/读写/元数据操作 | fs/fs_abstraction.h → walk() / read() / write() |
| CriteriaFilter | participant | 六维度文件筛选器（路径/类型/名称/mtime/大小/所有者） | filters/criteria_filter.h → apply() |
| Packer | participant | 归档打包器（Tar / Zip），将文件集合打包为单一归档 | pack/packer.h → pack() |
| Compressor | participant | 压缩器（gzip / zstd / lzma），对归档文件进行压缩 | compress/compressor.h → compress() |
| Encryptor | participant | 加密器（AES256 / SM4），对归档文件进行加密 | crypto/encryptor.h → encrypt() |
| LocalStorage | participant | 本地存储实现，提供文件写入能力 | storage/local_storage.h → writeFile() |
