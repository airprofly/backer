# 图名

任务状态转换图——备份任务生命周期

## 概述

本状态图描述了 Backer 备份还原工具中一次完整备份任务的生命周期，从任务创建开始，经历遍历扫描、筛选过滤、打包/目录镜像写入、压缩、加密、归档写入等阶段，最终到达任务完成或任务失败的终止状态。模型覆盖了归档模式和目录镜像模式两条主路径，以及压缩和加密两个可选后处理阶段，并标注了各阶段可能触发的异常条件与对应的错误码。

## 状态说明

| 状态名称 | 含义 | 进入条件 |
|----------|------|----------|
| 任务已创建 | BackupOptions 构建完成，任务对象初始化 | CLI11 参数校验通过（main.cpp: CLI11_PARSE） |
| 遍历扫描中 | BackupEngine 正在递归遍历源目录树 | engine.backup() 被调用且源路径验证通过（backup_engine.cpp:32-43） |
| 筛选过滤中 | CriteriaFilter 正在执行六维度匹配计算 | fs_->walk() 返回且 filter != nullptr（backup_engine.cpp:58-60） |
| 打包中（复合） | Packer 正在将文件集合打包为归档流 | CriteriaFilter::apply() 完成且 packer != nullptr |
| 目录镜像写入中（复合） | 逐项将源文件复制到目标目录，恢复元数据 | fs_->walk() 返回且 filter == nullptr，或 filter->apply() 完成且 packer == nullptr |
| 压缩中 | Compressor 正在对归档文件进行压缩 | buildCompressor 成功创建（commands.cpp:574-589） |
| 加密中 | Encryptor 正在对归档文件进行加密 | buildEncryptor 成功创建，密码已解析（commands.cpp:592-614） |
| 归档写入中 | 最终文件（压缩/加密后）落盘确认 | 打包/压缩/加密阶段完成，写操作开始 |
| 任务完成 | 备份成功，统计信息输出 | 写入成功，统计信息组装完毕（commands.cpp:616-625） |
| 任务失败 | 备份过程出现不可恢复错误 | 任意阶段检测到异常条件 |

## 异常路径

| 源状态 | 异常条件 | 错误码 | 对应代码位置 |
|--------|----------|--------|-------------|
| 任务已创建 | 源路径不存在或不是目录 | ErrorCode::kPathNotFound | backup_engine.cpp:32-43 |
| 遍历扫描中 | walk 权限不足或路径不可读 | ErrorCode::kPermissionDenied | backup_engine.cpp:47-51 |
| 筛选过滤中 | 筛选处理异常 | — | criteria_filter.cpp |
| 目录镜像写入中 | 磁盘空间不足，文件写入失败 | ErrorCode::kDiskFull | backup_engine.cpp:79-82, 165-169 |
| 打包中 | 打包报错（文件读取失败等） | ErrorCode::kPackError | backup_engine.cpp:68-72 |
| 压缩中 | 压缩算法异常，transformFile 返回 false | — | commands.cpp:583-586 |
| 加密中 | 加密失败，密码未提供或密钥错误 | — | commands.cpp:600-613（resolvePassword 为空或 encryptFile 失败） |

所有异常路径的终点均为「任务失败」状态，该状态会输出错误信息到 stderr 并以退出码 1 终止进程。
